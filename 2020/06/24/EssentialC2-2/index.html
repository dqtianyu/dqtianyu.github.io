Essential 第二章第二小节 笔记

调用函数

调用函数时候，会在内存中建立一个“程序堆栈”，在函数内定义的对象称为局部对象（local object），函数执行完毕，内存就会被释放掉，里面的内容会被弃置。

函数传值的三种方式

传值 pass by value

这种方式会将对象复制一份，传到函数内，函数内是一个新的对象。

传址 pass by reference

一个传址的例子： 

int main()
{
    int a = 1;
    change(a);
    cout<<a<<endl;
    system("pause");
}
void change(int &a){
    a = 2;
}
  1. C++不允许改变reference所代表的对象，它们必须从一而终。
    下面这个例子中，c作为a的引用，在c=b后，改变的也仅仅是a的值，无法对c再进行赋值。
int a = 2;
int b = 3;
int &c = a;
cout<<c<<endl;
c = b;
cout<<a<<endl;
  1. 对reference的所有操作，等同于对reference所代表的对象操作。
  2. 会降低复制大型对象的额外负担。

传址 pointer

pointer和refererce在传递方面效果相同，都是传递地址，但用法不同，一个是”->”，另一个是”*vec”。

int main()
{
    int ia[8]={8,32,3,13,1,21,5,2};
    vector<int> vec(ia,ia+8);
    display(&vec);

    system("pause");
}
void display(const vector<int> *vec)
{
    if(!vec)
    {
        cout<<"display():the vector pointer is 0 \n";
        return;
    }
    for(int ix=0;ix<vec->size();ix++)
        cout<<(*vec)[ix]<<' ';
    cout<<endl;
}
  1. pointer 传址应先确定指针非0。
  2. 除非为了更改值，否则传递内置类型，应使用pass by value的方式，而非传址。
  3. 这里在传递参数时候用了 const 这样传递的值不可以改变。

作用域及范围

函数执行完毕，这块区域内容会被弃置，返回的是对象的副本 。
上述针对local scope 而file scope 则不受限制,file scope定义的对象具有 static extent。

初始化问题

老版本的C++（03或98 存疑）不会在local scope使内置对象初始化比如在main函数中，int a ，此时a不会初始化为0，然而新版本（11、14）不会出现这个现象？不过为了确保万无一失，还是初始化好了。

内存的动态管理（堆内存）

int *a;
a = new int;

其内存是由程序的空闲空间(free store)分配而来，也被称为 heap memory(堆内存)
分配通过new 释放通过 delete
一直不释放的话，称之为内存泄漏（memory leak）
